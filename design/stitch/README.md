# Stitch Design References

Generated Stitch files are implementation references, not product authority. Resolve behavior
against `docs/PRD.md` and `docs/UX_FLOWS.md`, visual rules against `docs/DESIGN_SYSTEM.md`, and
security against `docs/SECURITY_PRIVACY.md`.

Indexes:

- `screens.json` — six canonical app and workout-detail references;
- `onboarding/screens.json` — sixteen authentication, eligibility, onboarding, HealthKit, and
  initial-plan state references; and
- `account/screens.json` — Account/Profile references and outstanding states.

Every imported screen folder contains generated HTML, a Stitch screenshot, and local metadata.
Generated code must be translated into Flutter components; it must not be copied as product logic or
treated as authoritative behavior.
